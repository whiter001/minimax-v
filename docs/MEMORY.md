# Memory & Context Persistence (内存与上下文持久化)

本系统提供了多种方式来维持跨会话的记忆和上下文。除了静态的 Skill 和 SOP 外，`session_note` 和 `experience` 是最主要的动态记忆工具。

## Session Note (会话笔记)

`session_note` 是一个高度动态的、面向项目的“短期记忆”工具。它旨在解决 AI 在开启新会话时失去前序任务进度和临时决策的问题。

### 核心定位

- **项目状态机**：记录此时此刻的进度、Bug 状态、临时决策。
- **任务看板**：类似于跨会话的 `TODO.md`，用于“续接断点”。
- **临时约束**：记录仅针对当前项目的编码风格或流程偏好。

### 工具操作

AI 可以通过以下操作管理笔记：

- `read`: 读取当前所有笔记。
- `write`: 覆盖写入（替换全部内容）。
- `append`: 追加内容到现有笔记。

### 存储路径

- `~/.config/minimax/session_notes.md`

### 最佳实践

1. **任务结束时总结**：在调用 `task_done` 之前，通过 `session_note(action: "append")` 记录核心结论和未完成事项。
2. **启动时同步**：新会话开始时优先 `read` 笔记，以对齐项目背景。
3. **定期清理**：当笔记过长或过时，执行 `write` 进行重写和精简。

## Experience (经验总结)

与 `session_note` 不同，`experience` 旨在将通用的、可复用的“教训”沉淀为长期知识。

- **差异性**：如果一个 Bug 的处理技巧适用于所有 V 语言项目，记录到 `experience`。如果它仅适用于当前项目的特定逻辑，记入 `session_note`。
- **自动检索**：`experience` 写入知识库后，AI 在后续会话中会根据语义相似度自动检索并载入相关上下文。

## 综合工作流建议

1. **短期(本周)**: 使用 `session_note` 进行进度追踪。
2. **中期(本项目)**: 复杂的逻辑决策记入 `session_note`。
3. **长期(全平台)**: 验证有效的模式和最佳实践通过 `record_experience` 固化。
